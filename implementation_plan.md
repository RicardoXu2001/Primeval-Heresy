# 🏰 古堡黑杰克 - Roguelike 21点卡牌游戏 Demo

## 概述

基于用户描述的游戏概念，创建一个可运行的网页端 Demo，核心玩法为 **21点 + Roguelike + Build Diversity**，融入克苏鲁/哥特美学风格。

## 核心设计

### 游戏流程
```
大厅 → 选择房间 → 21点对战(使徒) → 胜利获得印记 → 下一个房间 → ... → 规则之房(Boss)
                                    ↓
                              特殊胜利条件 → 进入指定房间(True Path)
```

### 技术方案
- **纯前端**: HTML + CSS + JavaScript（单文件或少量文件）
- **无需后端**: 所有游戏逻辑在客户端运行
- **风格**: 深色哥特/克苏鲁风格，带有神秘感的UI

## 游戏系统设计

### 1. 21点核心规则
- 标准21点规则：A可算1或11，JQK算10
- **Hit（要牌）** / **Stand（停牌）** / **Double（加倍）**
- 爆牌(>21)判负，比大小决胜
- 使徒(AI)按照不同策略进行要牌（不同使徒不同AI策略）

### 2. 房间系统（Roguelike）
- **4个时代房间** + **1个Boss房间（规则之房）**
  - 房间A：1920年代 - "舞会大厅"（使徒：堕落的贵族）
  - 房间B：1500年代 - "炼金工坊"（使徒：疯狂炼金术士）
  - 房间C：1800年代 - "图书馆"（使徒：沉默的学者）
  - 房间D：古代 - "祭坛"（使徒：远古祭司）
  - 规则之房：古神
- **Normal路线**：随机顺序挑战房间，击败4个使徒后进入Boss房
- **True路线**：在每个房间达成特殊胜利条件，按 A→B→C→D 顺序挑战

### 3. 特殊胜利条件（进入True Path的关键）
- 房间A：以恰好21点获胜（Blackjack）
- 房间B：以5张牌不爆获胜（五龙）
- 房间C：以刚好比对手多1点获胜（险胜）
- 房间D：连续3局获胜（完美碾压）

### 4. 印记系统（本局增益 - Roguelike海克斯）
每次胜利后从3个随机印记中选择1个，仅在当前一轮游戏(run)中有效：
- **偷窥之眼**：可以看到庄家的暗牌
- **命运之手**：每局可重抽一张牌
- **不屈意志**：爆牌阈值+2（23点才爆）
- **双面硬币**：A始终算11但不会爆牌
- **暗影护盾**：首次爆牌免死
- **贪婪之触**：获胜时额外获得1个印记选择
- **时间回溯**：每局可撤销一次要牌
- **混沌印记**：随机增减对手点数1-3

### 5. 遗物系统（永久Build - BD元素）
通过多次游戏(run)积累的经验值解锁，永久生效：

**攻击流派：**
- 锐利之刃：初始手牌+1张
- 暗杀之心：对手17点以上时，你的点数+2

**防御流派：**
- 坚韧之甲：爆牌阈值永久+1
- 生命之泉：每局多一次要牌后反悔的机会

**控制流派：**
- 洞察之眼：可看到牌堆下一张牌
- 束缚之链：对手首次要牌时有30%几率获得最小点数牌

### 6. Meta元素
- 多次失败后古神的台词会发生变化（嘲讽 → 好奇 → 恐惧）
- True Ending后解锁新的开场文字
- 运行次数(run count)会被记录并影响对话
